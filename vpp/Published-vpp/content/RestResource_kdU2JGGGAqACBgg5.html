REST Service - Add questionnaire link

Properties
Name Value
Description

Aνέβασμα αρχείου JSON με δεδομένα ενός νέου ερωτηματολογίου.

Στην http κλήση το αρχείο πρέπει να είναι κωδικοποιημένο ως πεδίο "file" σε multipart/form-data κωδικοποίηση.

Uri /admin/questionnaire_upd
Method POST
Resource Type Service Provider

Parameters Summary
Name In Type Required Description
file formData true

Το ερωτηματολόγιο ως JSON αρχείο.

Η απαιτούμενη δομή του αρχείου JSON είναι η ακόλουθη:

Πεδίο Τύπος Περιγραφή
questionnaireID String Το αναγνωριστικό του ερωτηματολογίου
questionnaireTitle String Ο τίτλος του ερωτηματολογίου
keywords List Λίστα με λέξεις-κλειδιά. Κάθε λέξη-κλειδί είναι ένα string
questions List Λίστα με ερωτήσεις. Για κάθε ερώτηση, ένα JSON object που περιέχει τα ακόλουθα (qID, qtext, required, type, options) :
    qID String Το αναγνωριστικό της ερώτησης
    qtext String Το κείμενο της ερώτησης
    required String "TRUE" ή "FALSE"
    type String "question" ή "profile"
    options List Λίστα με τις επιλογές. Για κάθε επιλογή, ένα JSON object με τα ακόλουθα (optID, opttxt, nextqID) :
         optID String Το αναγνωριστικό της απάντησης
         opttxt String Το κείμενο της απάντησης
         nextqID String Το αναγνωριστικό της επόμενης ερώτησης

 

 

Για το αρχείο JSON πρέπει επιπλέον να ισχύουν τα ακόλουθα:

  • Το αναγνωριστικό της κάθε ερώτησης πρέπει να είναι μοναδικό.
  • Το αναγνωριστικό μιας ερώτησης δεν μπορεί να είναι "-".
  • Για όλες τις επιλογές της τελευταίας ερώτησης, το αναγνωριστικό της επόμενης ερώτησης πρέπει να είναι "-".
  • Για όλες τις επιλογές οποιασδήποτε ερώτησης, το αναγνωριστικό της επόμενης ερώτησης πρέπει να αντιστοιχεί σε ερώτηση που είναι πιο μετά στη λίστα.
  • Οι ερωτήσεις πρέπει να είναι ταξινομημένες ως προς το αναγνωριστικό της ερώτησης.
  • Για οποιαδήποτε ερώτηση, το αναγνωριστικό της κάθε επιλογής πρέπει να είναι μοναδικό.
  • Για οποιαδήποτε ερώτηση, οι επιλογές πρέπει να είναι ταξινομημένες ως προς το αναγνριστικό της επιλογής.
